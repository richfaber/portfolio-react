# i18n 다국어 처리

## 구조

```
src/i18n/
  index.ts          # Context API 버전 초기화
  index.react.ts    # react-i18next 버전 초기화
  locales/
    ko.json         # 한국어 번역
    en.json         # 영어 번역
src/context/
  I18nContext.tsx   # Context API 버전 Provider/Hook
src/page/test/
  I18n.tsx          # Context API 버전 예제
  I18nReact.tsx     # react-i18next 버전 예제
```

---

## 예제 1: Context API + i18next

react-i18next 없이 Context API로 직접 Provider와 Hook을 구현한 버전.
i18next는 번역 엔진으로만 사용하고, React 연결은 직접 처리.

### 초기화 (`src/i18n/index.ts`)

```ts
import i18next from 'i18next'

i18next.init({
  resources: { ko: { translation: ko }, en: { translation: en } },
  lng: 'ko',
  fallbackLng: 'ko',
  interpolation: { escapeValue: false }
})

export default i18next
```

### 사용

```tsx
import { useI18n } from '@/context/I18nContext'

const { t, locale, setLocale } = useI18n()

t('home.title')
t('home.greeting', { name: '철수' })  // 변수 치환
setLocale('en')
```

### 주의: React Compiler 이슈

`t` 함수에 `lng: locale` 을 명시적으로 넘겨야 React Compiler가 의존성을 추적함.
넘기지 않으면 언어가 바뀌어도 화면이 갱신되지 않음. → [자세히](limitations.md)

```ts
// I18nContext.tsx
t: (key, options?) => i18next.t(key, { lng: locale, ...options })
```

## 예제 2: react-i18next

react-i18next가 Context API 연결을 자동으로 처리하는 버전.

### 초기화 (`src/i18n/index.react.ts`)

```ts
import { initReactI18next } from 'react-i18next'

i18next
  .use(initReactI18next)  // 이 한 줄이 Context API 연결을 자동으로 처리
  .init({ ... })
```

### 사용

```tsx
import { useTranslation } from 'react-i18next'

const { t, i18n } = useTranslation()

t('home.title')
i18n.changeLanguage('en')
i18n.language  // 현재 언어
```

별도 Provider 등록 불필요. `index.react.ts` 를 import 하는 것만으로 동작.

---

## 두 방식 비교

| | Context API + i18next | react-i18next |
|---|---|---|
| 설치 | `i18next` | `i18next` + `react-i18next` |
| Provider 등록 | 직접 구현 필요 | 불필요 |
| 훅 | `useI18n()` (직접 구현) | `useTranslation()` |
| 언어 변경 | `setLocale('en')` | `i18n.changeLanguage('en')` |
| React Compiler 대응 | 수동으로 `lng: locale` 명시 | 라이브러리가 자동 처리 |
| 적합한 상황 | Context API 학습 목적 | 실무 |

## 번역 파일 구조

```json
{
  "home": {
    "title": "홈",
    "greeting": "안녕하세요, {{name}}님!"
  }
}
```

변수 치환은 `{{변수명}}` 문법 사용.
